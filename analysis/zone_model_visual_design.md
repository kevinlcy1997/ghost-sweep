# Zone Model Visual Design Philosophy

The visual language should feel like a carefully drafted operations map: precise, quiet, and deeply intentional. It should avoid decorative noise and instead use layered structure, subtle color coding, and measured spacing to make a complicated pipeline feel graspable. The composition should look meticulously crafted, with every line and label placed as if a senior information designer labored over it with painstaking attention.

The aesthetic should evoke Hong Kong as a dense road-and-district system without drawing a literal tourist map. Use small hexagonal geometry, routed connector lines, and a restrained civic palette: ink, mist, harbor blue, signal amber, and alert green. The result should feel technical but not sterile, the product of deep expertise rather than a generic machine-learning diagram.

Hierarchy matters more than decoration. The viewer should understand the story in one pass: raw alerts become zones, zones become time-window training rows, models rank near-term risk, and the output is a top-zone forecast for map display. Key metrics should appear as confident instrument readings, not marketing claims. The work should feel master-level in its restraint.

The final piece should be pristine enough to sit beside an experiment report: polished, legible, and useful. It should make sparse geographic prediction feel concrete by showing where the target comes from, why ranking metrics matter, and how the result should be interpreted operationally.
